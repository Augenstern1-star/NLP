  # 中文自然文本标注数据规范 V1.0
  ## 一、总体目标
  本规范用于对中文自然文本进行人工辅助标注。各位同学可以自行调用任意大语言模型辅助完成标注，但最终提交的数据必须采用统一 JSON 格式。
  提交数据将用于后续的分组作业。
  因此，标注结构应尽量对应《地书》标注系统中的两类任务（**每人只需选本人分配到的《地书》标注任务一致的两项任务类型**）：
  1. **词标注任务类型**
     - `[Multi] 词性/符号性质 / POS-like Category`
     - `[Multi] 类形态特征 / Morphological-like Features`
     - `[Multi] 语义基元 / Semantic Primitives`
     - `[Text] 语境实际含义 / Pragmatic Meaning`
     - `[Text] 字面对应词 / Literal Gloss`
  2. **句标注任务类型**
     - `[Single] 与前一组的篇章关系 / Discourse Relation`
     - `[Text] 自由翻译 / Free Translation`
  ***
  # 二、提交文件基本要求
  每位学生提交一个 JSON 文件。
  文件名建议为：
  ```
  学号_姓名_中文文本标注.json
  ```
  例如：
  ```
  20240123_张三_中文文本标注.json
  ```
  JSON 文件编码必须为：
  ```
  UTF-8
  ```
  JSON 文件最外层必须是一个对象 `{}`，不得直接提交数组 `[]`。
  ***
  # 三、JSON 顶层结构
  每份 JSON 文件应包含以下 5 个一级字段：
  ```json
  {
    "project": {},
    "student": {},
    "source_text": {},
    "units": [],
    "annotations": []
  }
  ```
  各字段含义如下：
  | 字段          | 类型   | 是否必填 | 含义                                       |
  | ------------- | ------ | -------- | ------------------------------------------ |
  | `project`     | object | 必填     | 项目与任务信息                             |
  | `student`     | object | 必填     | 学生信息                                   |
  | `source_text` | object | 必填     | 被标注的中文自然文本信息                   |
  | `units`       | array  | 必填     | 文本切分后的基本单位，包括词、短语、句子等 |
  | `annotations` | array  | 必填     | 对各单位的标注结果                         |
  ***
  # 四、`project` 字段规范
  ```json
  "project": {
    "project_name": "《地书》与自然语言文本差异性分析",
    "data_type": "natural_language_text",
    "schema_version": "1.0",
    "annotation_language": "zh-CN",
    "created_at": "2026-04-26T20:30:00+08:00",
    "llm_assisted": true,
    "llm_tools": ["ChatGPT", "DeepSeek"]
  }
  ```
  说明：
  | 字段                  | 含义                               |
  | --------------------- | ---------------------------------- |
  | `project_name`        | 固定填写项目名称                   |
  | `data_type`           | 固定为 `"natural_language_text"`   |
  | `schema_version`      | 当前版本填写 `"1.0"`               |
  | `annotation_language` | 标注语言，中文填写 `"zh-CN"`       |
  | `created_at`          | 文件创建时间                       |
  | `llm_assisted`        | 是否使用大语言模型辅助             |
  | `llm_tools`           | 使用过的大语言模型名称，可填写多个 |
  如果未使用大语言模型，则填写：
  ```json
  "llm_assisted": false,
  "llm_tools": []
  ```
  ***
  # 五、`student` 字段规范
  ```json
  "student": {
    "student_id": "20240123",
    "student_name": "张三",
    "assigned_section": "第3组文本",
    "word_task_type": "semantic_primitives",
    "sentence_task_type": "free_translation"
  }
  ```
  说明：
  | 字段                 | 含义                   |
  | -------------------- | ---------------------- |
  | `student_id`         | 学号                   |
  | `student_name`       | 姓名                   |
  | `assigned_section`   | 分配到的文本或章节编号 |
  | `word_task_type`     | 分配到的词标注任务类型 |
  | `sentence_task_type` | 分配到的句标注任务类型 |
  其中 `word_task_type` 只能从以下值中选择：
  ```json
  [
    "pos_like_category",
    "morphological_like_features",
    "semantic_primitives",
    "pragmatic_meaning",
    "literal_gloss"
  ]
  ```
  `sentence_task_type` 只能从以下值中选择：
  ```json
  ["discourse_relation", "free_translation"]
  ```

  # 六、`source_text` 字段规范
  ```json
  "source_text": {
    "text_id": "text_001",
    "title": "示例文本",
    "source": "教师提供",
    "raw_text": "今天下午，小明走进图书馆，发现窗边的座位已经被占满了。于是他转身去了自习室。",
    "notes": "用于自然语言文本标注示例"
  }
  ```
  说明：
  | 字段       | 含义                                         |
  | ---------- | -------------------------------------------- |
  | `text_id`  | 文本编号                                     |
  | `title`    | 文本标题                                     |
  | `source`   | 文本来源，如“教师提供”“教材节选”“自选文本”等 |
  | `raw_text` | 完整原文                                     |
  | `notes`    | 可选说明                                     |

  # 七、`units` 字段规范
  `units` 用于记录文本被切分后的基本单位。为了和《地书》系统中的 Atom 和 Group 对应，建议把中文文本也分为两类单位：
  1. **word 单位**：词、短语、符号或最小可解释片段；
  2. **sentence 单位**：句子或小句。
  ## 7.1 基本结构
  ```json
  "units": [
    {
      "unit_id": "sent_001",
      "unit_type": "sentence",
      "text": "今天下午，小明走进图书馆，发现窗边的座位已经被占满了。",
      "sentence_index": 1,
      "char_start": 0,
      "char_end": 28,
      "children": ["word_001", "word_002", "word_003"]
    },
    {
      "unit_id": "word_001",
      "unit_type": "word",
      "text": "今天下午",
      "sentence_id": "sent_001",
      "word_index_in_sentence": 1,
      "char_start": 0,
      "char_end": 4
    }
  ]
  ```
  ## 7.2 字段说明
  | 字段                     | 类型   | 是否必填     | 含义                     |
  | ------------------------ | ------ | ------------ | ------------------------ |
  | `unit_id`                | string | 必填         | 单位唯一编号             |
  | `unit_type`              | string | 必填         | `"word"` 或 `"sentence"` |
  | `text`                   | string | 必填         | 该单位对应的文本         |
  | `sentence_index`         | number | 句子必填     | 句子在全文中的序号       |
  | `sentence_id`            | string | 词单位必填   | 该词所属句子的编号       |
  | `word_index_in_sentence` | number | 词单位必填   | 词在句中的序号           |
  | `char_start`             | number | 建议填写     | 在原文中的起始字符位置   |
  | `char_end`               | number | 建议填写     | 在原文中的结束字符位置   |
  | `children`               | array  | 句子建议填写 | 该句包含的词单位 ID      |
  ***
  # 八、`annotations` 字段规范
  `annotations` 是核心字段，用于保存标注结果。每条标注对应一个单位和一个任务。
  基本结构如下：
  ```json
  {
    "annotation_id": "ann_001",
    "target_id": "word_001",
    "target_type": "word",
    "task_id": "semantic_primitives",
    "task_label_zh": "语义基元",
    "task_label_en": "Semantic Primitives",
    "value": ["time", "setting"],
    "confidence": 0.85,
    "llm_used": true,
    "llm_name": "ChatGPT",
    "annotator_note": "“今天下午”表示事件发生的时间背景。"
  }
  ```
  ## 8.1 字段说明
  | 字段             | 类型           | 是否必填 | 含义                     |
  | ---------------- | -------------- | -------- | ------------------------ |
  | `annotation_id`  | string         | 必填     | 标注记录唯一编号         |
  | `target_id`      | string         | 必填     | 被标注单位 ID            |
  | `target_type`    | string         | 必填     | `"word"` 或 `"sentence"` |
  | `task_id`        | string         | 必填     | 标注任务编号             |
  | `task_label_zh`  | string         | 必填     | 中文任务名称             |
  | `task_label_en`  | string         | 必填     | 英文任务名称             |
  | `value`          | string / array | 必填     | 标注值                   |
  | `confidence`     | number         | 建议填写 | 标注信心，范围 0–1       |
  | `llm_used`       | boolean        | 必填     | 是否使用大语言模型辅助   |
  | `llm_name`       | string / null  | 必填     | 使用的大语言模型名称     |
  | `annotator_note` | string         | 可选     | 标注者说明               |
  ***
  # 九、七类任务的具体填写规范
  ## 9.1 `[Multi] 词性/符号性质 / POS-like Category`
  ### task_id
  ```json
  "pos_like_category"
  ```
  ### value 类型
  数组 `array`
  ### 推荐选项
  ```json
  [
    "noun_like",
    "verb_like",
    "adjective_like",
    "adverb_like",
    "pronoun_like",
    "numeral_like",
    "classifier_like",
    "preposition_like",
    "conjunction_like",
    "particle_like",
    "punctuation_or_symbol",
    "other"
  ]
  ```
  ### 示例
  ```json
  {
    "annotation_id": "ann_001",
    "target_id": "word_004",
    "target_type": "word",
    "task_id": "pos_like_category",
    "task_label_zh": "词性/符号性质",
    "task_label_en": "POS-like Category",
    "value": ["verb_like"],
    "confidence": 0.95,
    "llm_used": true,
    "llm_name": "ChatGPT",
    "annotator_note": "“走进”表示动作，因此标为动词类。"
  }
  ```
  ***

  ## 9.4 `[Text] 语境实际含义 / Pragmatic Meaning`
  ### task_id
  ```json
  "pragmatic_meaning"
  ```
  ### value 类型
  字符串 `string`
  ### 填写要求
  填写该词在当前上下文中的实际含义，而不是孤立词典义。
  ### 示例
  ```json
  {
    "annotation_id": "ann_004",
    "target_id": "word_010",
    "target_type": "word",
    "task_id": "pragmatic_meaning",
    "task_label_zh": "语境实际含义",
    "task_label_en": "Pragmatic Meaning",
    "value": "表示座位已经没有可供小明使用的位置，暗示他需要改变原计划。",
    "confidence": 0.88,
    "llm_used": true,
    "llm_name": "ChatGPT",
    "annotator_note": "这里的“占满”不仅描述状态，也推动后续行为。"
  }
  ```
  ***
  ## 9.5 `[Text] 字面对应词 / Literal Gloss`
  ### task_id
  ```json
  "literal_gloss"
  ```
  ### value 类型
  字符串 `string`
  ### 填写要求
  填写最接近字面意义的对应解释，尽量简短。
  ### 示例
  ```json
  {
    "annotation_id": "ann_005",
    "target_id": "word_004",
    "target_type": "word",
    "task_id": "literal_gloss",
    "task_label_zh": "字面对应词",
    "task_label_en": "Literal Gloss",
    "value": "走入；进入",
    "confidence": 0.95,
    "llm_used": false,
    "llm_name": null,
    "annotator_note": ""
  }
  ```
  ***
  ## 9.6 `[Single] 与前一组的篇章关系 / Discourse Relation`
  ### task_id
  ```json
  "discourse_relation"
  ```
  ### value 类型
  字符串 `string`
  ### 推荐选项
  ```json
  [
    "start",
    "continuation",
    "sequence",
    "cause",
    "result",
    "contrast",
    "elaboration",
    "condition",
    "purpose",
    "turning_point",
    "summary",
    "other"
  ]
  ```
  ### 填写说明
  该任务用于句子单位。第一个句子一般填写：
  ```json
  "start"
  ```
  后续句子填写它与前一句或前一组句子的篇章关系。
  ### 示例
  ```json
  {
    "annotation_id": "ann_006",
    "target_id": "sent_002",
    "target_type": "sentence",
    "task_id": "discourse_relation",
    "task_label_zh": "与前一组的篇章关系",
    "task_label_en": "Discourse Relation",
    "value": "result",
    "confidence": 0.9,
    "llm_used": true,
    "llm_name": "ChatGPT",
    "annotator_note": "前一句说明图书馆座位已满，后一句说明小明因此转去自习室。"
  }
  ```
  ***
  ## 9.7 `[Text] 自由翻译 / Free Translation`
  ### task_id
  ```json
  "free_translation"
  ```
  ### value 类型
  字符串 `string`
  ### 填写要求
  对句子进行自然、完整、通顺的解释或翻译。由于原文就是中文，这里的“自由翻译”可理解为：
  > 用更清楚、自然的中文重述该句含义。
  ### 示例
  ```json
  {
    "annotation_id": "ann_007",
    "target_id": "sent_001",
    "target_type": "sentence",
    "task_id": "free_translation",
    "task_label_zh": "自由翻译",
    "task_label_en": "Free Translation",
    "value": "今天下午，小明去了图书馆，但发现靠窗的位置都已经有人坐了。",
    "confidence": 0.92,
    "llm_used": true,
    "llm_name": "ChatGPT",
    "annotator_note": "保留原句基本信息，同时使表达更自然。"
  }
  ```

  # 十一、填写时的具体要求
  ## 1. 不要改变字段名称
  例如必须写：
  ```json
  "target_id"
  ```
  不要写成：
  ```json
  "目标ID"
  "targetId"
  "id_of_target"
  ```
  否则后续程序难以自动汇总。
  ***
  ## 2. 每个学生只填写自己被分配的任务
  例如某学生被分配到：
  ```json
  "word_task_type": "semantic_primitives",
  "sentence_task_type": "free_translation"
  ```
  那么他的 `annotations` 中应主要包含：
  ```json
  "task_id": "semantic_primitives"
  ```
  和：
  ```json
  "task_id": "free_translation"
  ```
  不要额外混入大量其他任务，除非教师特别要求。
  ***
  ## 3. `word` 单位和 `sentence` 单位要区分清楚
  词标注任务只能标注：
  ```json
  "target_type": "word"
  ```
  句标注任务只能标注：
  ```json
  "target_type": "sentence"
  ```
  不要把句子标注任务写到词上，也不要把词标注任务写到句子上。
  ***
  ## 4. 多选任务的 `value` 必须是数组
  正确：
  ```json
  "value": ["action", "motion"]
  ```
  错误：
  ```json
  "value": "action, motion"
  ```
  ***
  ## 5. 文本任务的 `value` 必须是字符串
  正确：
  ```json
  "value": "表示该人物改变了原来的行动路线。"
  ```
  错误：
  ```json
  "value": ["表示该人物改变了原来的行动路线。"]
  ```
  ***
  ## 6. 单选任务的 `value` 必须是一个字符串
  正确：
  ```json
  "value": "result"
  ```
  错误：
  ```json
  "value": ["result"]
  ```
  ***
  ## 7. 保留大语言模型使用信息
  如果标注时调用了大语言模型，必须如实填写：
  ```json
  "llm_used": true,
  "llm_name": "ChatGPT"
  ```
  如果没有调用，则填写：
  ```json
  "llm_used": false,
  "llm_name": null
  ```
  这不是为了扣分，而是为了后续分析：
  - 人工标注与 AI 辅助标注是否有差异；
  - 不同模型辅助下的标注结果是否更一致；
  - 哪些任务更依赖大语言模型。
  ***

  | 单选任务的 `value` 是字符串                                                   | □        |
  | 如实填写了是否使用大语言模型                                                  | □        |
